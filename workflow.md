### LLM 康复推荐整体流程

**任务目标:** 设计并实现一个基于LLM的“一站式康复推荐方案”生成系统。系统需接收患者的多模态数据（结构化文本、医学影像、交互补充信息），自动完成**疾病诊断 → 药物与治疗方案生成 → 排班整合**，最终输出完整的康复推荐方案。

**核心流程与功能模块描述:**

**阶段一：多模态数据输入与预处理**

1. 输入接收 (病人数据):
   - **数据源:** 系统接收来自患者的多通道数据。
   - 数据类型:
     - **结构化/非结构化文本:** 病历、主诉、现病史、既往史、检查报告文本等。
     - **医学影像数据 (图像数据):** 如png、MRI、CT等。
     - **交互式补充信息:** 通过用户交互界面（`补充输入工具及交互式输入`）获取的额外信息或对模糊点的澄清。
2. 数据预处理与特征提取:
   - 图像处理模块:
     - **功能:** 调用 `图像解析接口及工具库`对输入的 `图像数据`进行解析。
     - **处理:** 包括图像分割、特征提取、病灶识别等。
     - **输出:** 结构化的图像分析结果或关键特征。
   - 文本与交互信息处理模块:
     - **功能:** 处理 `文本数据`和 `补充输入工具及交互式输入`提供的信息。
     - **处理:** 利用 `结构化信息、词表、规范`（如医学术语词典、本体库）进行信息提取、标准化、实体识别（如症状、体征、药品）、关系抽取。
     - **输出:** 结构化的、标准化的患者信息表示。
   - **数据融合:** 将来自图像、文本、交互信息的处理结果整合成统一的 `诊断前置信息`。
   - **数据流:** 原始多模态数据 → 各自预处理模块 → `诊断前置信息`。

**阶段二：疾病诊断 (含交互式 уточнение - clarification)**

1. 初步诊断:
   - **输入:** `诊断前置信息`，`专家知识库(结构化数据)`。
   - **功能模块:** `诊断阶段工具`（可基于LLM的推理能力，结合知识库）。
   - **处理:** 根据输入的患者信息和医学知识库，生成初步的疾病诊断或鉴别诊断列表。
   - **输出:** 初步诊断结果。
2. 交互式信息补充与诊断精炼 (决策点):
   - **判断:** 系统评估当前信息是否足以做出明确诊断 (`是否需要多轮补充信息`)。
   - 是 (Yes):
     - **交互:** 系统生成 `反馈诊断`信息（如可能的诊断方向、需要澄清的问题），通过 `补充输入工具及交互式输入`与用户（医生或患者）交互，获取额外信息。
     - **数据流:** 反馈诊断 -> 交互输入 -> 更新后的 `诊断前置信息` -> 返回步骤1进行再诊断。
   - 否 (No):
     - **输出:** 确认最终的疾病诊断结果。
   - **数据流:** 初步诊断 -> [可选交互循环] -> 最终诊断结果。

**阶段三：药物与治疗方案生成**

1. 方案生成:
   - **输入:** 最终诊断结果，`专家知识库(结构化数据)`，患者详细信息（来自 `诊断前置信息`），`业务规则引擎`（包含治疗指南、禁忌症、本地化策略等）。
   - 功能模块:
     - **药物推荐模块 (`药物推荐工具`):** 基于诊断、患者特征（年龄、过敏史、肝肾功能等）、知识库和业务规则，生成 `药物推荐`列表（包括药品名称、剂量、用法、疗程）。
     - **治疗计划生成模块 (`治疗计划生成工具集`):** 基于诊断、患者功能状态、康复目标、知识库和业务规则，生成非药物 `治疗计划`（如物理治疗、作业治疗、康复训练项目、频率、周期）。
   - **处理:** LLM结合专业工具和规则，分别生成药物和治疗方案的初步建议。
   - **输出:** `药物推荐`列表，`治疗计划`列表。
   - **数据流:** 最终诊断 + 知识库 + 患者信息 + 规则 -> 药物推荐模块 -> `药物推荐`。 最终诊断 + 知识库 + 患者信息 + 规则 -> 治疗计划模块 -> `治疗计划`。

**阶段四：排班整合**

1. 排班与资源协调:
   - **输入:** `治疗计划`列表。
   - **功能模块:** `排班生成工具`。
   - **处理:** 将各项治疗活动，结合（隐含的或显式输入的）可用资源信息（如治疗师时间、设备可用性、病房安排），整合成一个具体可行的时间表。
   - **输出:** `治疗计划排班`（包含时间和资源的具体安排）。
   - **数据流:** `治疗计划` -> 排班生成工具 -> `治疗计划排班`。

**阶段五：最终方案生成与输出**

1. 方案整合与格式化:
   - **输入:** `药物推荐`列表，`治疗计划`列表。
   - **功能模块:** `方案整合工具`。
   - **处理:** 将诊断结果、药物推荐、治疗计划以及具体的排班信息整合成一份结构清晰、易于理解的“一站式康复推荐方案”。
   - **输出:** 最终的 `生成推荐治疗方案`。
2. **结束:** 流程完成，输出最终方案。

**关键考量 (用于LLM微调或提示工程):**

- **知识库整合:** 如何有效利用 `专家知识库(结构化数据)`来指导LLM的诊断和推荐，减少幻觉，提高准确性。
- **多模态融合:** 如何让LLM有效理解和融合来自文本、图像和交互的不同信息源。
- **交互设计:** `反馈诊断`和 `补充输入`环节的交互逻辑需要清晰、高效。
- **工具调用:** LLM需要能够理解何时以及如何调用外部工具（如 `图像解析`、`药物推荐`、`排班生成`等）。
- **可解释性与安全性:** 输出的方案需要有一定的可解释性，并且遵循医疗安全规范。
